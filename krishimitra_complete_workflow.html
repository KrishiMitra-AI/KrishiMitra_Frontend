Frontend — everything the user sees and touches

Frontend = the app screen. The farmer opens your app on their phone — whatever they SEE is frontend. It is built with React + Vite + Tailwind CSS.

📁
React App (the whole frontend project)
One folder. All 6 feature screens live here.
⚙️
Setup & Tools
How to create the project, what you need installed
💡
Vite
Like a "fast engine" that runs React. Command: npm create vite@latest. Every time you save a file, the browser refreshes instantly.
💡
Tailwind CSS
Styling using class names directly in your HTML. Instead of writing CSS files, you write className="text-green-600 font-bold" right in the component.
💡
React Router
Handles navigation — when farmer taps "Price Forecast", React Router switches the screen without reloading the whole page. Install: npm install react-router-dom
💡
Recharts
A ready-made chart library. You just give it data, it draws a line graph. Install: npm install recharts. Used only on Price Forecast screen.
📱
6 Screens to Build
Each screen = one .jsx file = one React component
🌿
Disease Scanner
Camera/upload button → image preview box → result card showing disease name, severity bar (green/yellow/red), 3 treatment steps in Hindi. Speaker button reads result aloud.
📈
Price Forecast
Two dropdowns (crop + state) → big price number → Recharts line chart (7 days) → recommendation card ("sell now" / "wait"). Voice button reads advice.
🤝
Mandi Swipe
Farmer: form to list crop (photo + qty + price + GPS). Buyer: swipe cards left/right (like Tinder). Match screen with buyer name + phone.
🌦️
Weather Alert
Home screen cards. Color coded: red = urgent, yellow = caution, green = good. Cards like "Kal baarish hogi — pesticide mat chidko".
🌾
Crop Advisor
4 inputs (soil, water, budget, location) → 3 crop cards ranked by expected profit. Each card has crop name, income estimate, sowing date, water need.
💬
KisanBot Chat
Floating button (visible on all screens). WhatsApp-style chat. Type or tap mic → speech to text → AI reply appears → voice reads it aloud.
🔗
How Frontend Connects to Backend
Frontend talks to Java backend using fetch() — like sending a letter and waiting for a reply
📡
API calls using fetch()
Example: farmer uploads photo → React calls fetch("https://your-backend.render.com/api/disease/scan", method: POST, body: photo) → Java processes → React gets back JSON result → shows it on screen.
🔑
JWT Token (login)
After login, backend gives a "token" (a long secret code). Frontend saves it in memory and sends it with every request so backend knows who is talking.
⚡
Supabase Realtime (Mandi Swipe)
Frontend listens live to Supabase. When a buyer swipes right, database updates instantly → React shows "Ek khareedaar mila!" notification without refreshing page.
🌐
Web Speech API (voice)
FREE, built into Chrome/Android browser. Mic button → browser converts Hindi speech to text (no API key needed). Also reads text aloud using speechSynthesis.speak().

Backend — the brain behind the app

Backend = the server. When the farmer sends a photo, the frontend can't talk to Gemini AI directly (that would expose the API key). So frontend sends the photo to Java → Java calls Gemini → sends result back. Built with Java 17 + Spring Boot.

📁
Spring Boot Project (the whole backend)
One Java project. All routes, logic, database calls live here.
⚙️
Setup & Project Structure
How to create the project, what folders mean what
💡
start.spring.io
Go to this website → select Java 17, Spring Boot 3, add dependencies → click Generate → download a ready-made project ZIP. Extract and open in IntelliJ or VS Code.
📂
controller/ folder
API route handlers. Example: DiseaseController handles POST /api/disease/scan. This is the "door" — requests come in here.
📂
service/ folder
Business logic. Example: DiseaseService.java calls Gemini API, reads the response, formats it. This is the "brain" — actual work happens here.
📂
repository/ folder
Database queries. Example: DiseaseRepository.save(result) saves a scan to Supabase. This is the "memory" — talks to database.
📂
model/ folder
Data shapes. Example: ScanResult.java has fields: disease_name, severity, treatment. Like a template for what data looks like.
🔑
application.properties
Where you store secret keys: Gemini API key, Supabase URL + password, OpenWeatherMap key. NEVER commit this file to GitHub. Add it to .gitignore!
🛣️
14 API Routes (endpoints)
Frontend calls these URLs to get things done
🔐
POST /api/auth/register + /api/auth/login
Farmer registers with phone number. Login returns a JWT token. All other endpoints check this token.
🌿
POST /api/disease/scan
Receives photo + voice note → saves photo to Supabase Storage → calls Gemini Vision → saves result to DB → returns Hindi diagnosis JSON.
📈
GET /api/prices/{crop} + /api/prices/forecast/{crop}
First fetches live price from Agmarknet. Second gets 30-day history from Supabase, sends to Gemini, returns 7-day forecast + sell advice.
🤝
POST /api/listings + GET /api/listings + POST /api/match
Farmer creates listing. Buyer fetches nearby listings by GPS. Buyer swipes right → match saved → Supabase Realtime notifies farmer instantly.
🌦️
GET /api/weather
Fetches 7-day weather from OpenWeatherMap → sends to Gemini → returns 3 farming action alerts in Hindi.
🌾
POST /api/advisor
Receives soil+water+budget+location → Java also fetches local weather + current prices → sends all to Gemini → returns top 3 crop suggestions in Hindi.
💬
POST /api/kisanbot/chat
Receives question + full chat history → sends to Gemini with KisanBot system prompt → returns Hindi reply → saves to Supabase.
🧠
The Core Logic Flow (how every request works)
Same 4-step pattern used for every feature
1️⃣
Controller receives request
Frontend sends request to Java. Controller checks JWT token (is this a real logged-in user?). If valid, passes data to Service.
2️⃣
Service does the work
Calls Gemini API / Agmarknet / OpenWeatherMap. Parses the JSON response. Builds a clean result object.
3️⃣
Repository saves to database
Saves the result to Supabase (scan history, chat log, price cache, etc.).
4️⃣
Controller returns JSON
Returns clean JSON to React frontend. React reads it and shows the UI (disease card, price chart, chat reply, etc.).

How to Test Backend
4 ways to check if your Java APIs work
🟢
Run locally first
Run mvn spring-boot:run. Server starts on port 8080. Test with browser or Postman at http://localhost:8080/api/...
📬
Postman
A free app to send fake requests to your API. You can test POST /api/disease/scan by attaching a photo, setting headers, and seeing the JSON response — without needing the frontend to be ready.
🌍
CORS config (important!)
By default, Java will BLOCK requests from your React app (different URL). You must add CORS config in Java to allow requests from your Vercel domain. Without this, frontend can't talk to backend.
🚀
Deploy to Render.com
Push code to GitHub → connect repo to Render → it builds automatically. You get a public URL like https://krishimitra-api.onrender.com. Share this with P1 (frontend) to replace localhost.
Database — the memory of your app

Database = where everything is saved. When farmer registers, their info is saved here. When they scan a disease, the result is saved here. You're using Supabase (PostgreSQL) — it's like a smart Google Sheet in the cloud. Free, and it also gives you file storage and live updates.

🗄️
Supabase — 3 things in one
PostgreSQL database + file storage + realtime updates. All free.
📊
5 Tables (like 5 Excel sheets)
Each table stores one type of data
👤
users table
Stores: id, phone number, name, role (farmer/buyer), village name, GPS lat/lng, when they registered. Every user = 1 row.
🌾
crop_listings table
Stores: which farmer listed it, crop name, quantity (kg), minimum price, photo URL, GPS location, status (available/matched/sold). Used by Mandi Swipe feature.
🤝
matches table
Stores: which listing was matched, which buyer swiped right, which farmer owns it, status (pending/accepted/completed), when it happened.
🌿
disease_scans table
Stores: which farmer did the scan, photo URL, disease name found by Gemini, severity (low/medium/high), full treatment text in Hindi, when it was scanned.
💬
chat_history table
Stores: which farmer's chat, whether it's a user message or KisanBot reply, full message text in Hindi, timestamp. Needed so KisanBot remembers previous messages.
🖼️
Supabase Storage (file storage)
Stores photos — crop disease photos and Mandi Swipe listing photos
📸
How it works
Java uploads the photo to Supabase Storage → gets back a URL → saves that URL in the database table. React displays the photo using that URL. Like Google Drive but for your app.
🪣
Buckets to create
Create two buckets in Supabase dashboard: "disease-photos" and "crop-listings". Set them to public so photos are viewable without login.
⚡
Supabase Realtime (live updates)
Used for Mandi Swipe — farmer gets notified instantly when buyer matches
🔔
How it works
React "subscribes" to the matches table. When a new row is inserted (buyer swiped right), Supabase sends a live notification to the farmer's browser. No page refresh needed. Go to Supabase → Table Editor → matches → enable Realtime toggle.
🔗
How to connect Java to Supabase
3 simple steps
1️⃣
Get connection string from Supabase
Go to Supabase dashboard → Settings → Database → copy the "Connection string". It looks like jdbc:postgresql://db.xxx.supabase.co:5432/postgres
2️⃣
Add to application.properties
Paste the URL + username + password into your Spring Boot application.properties file. Java will automatically connect on startup.
3️⃣
Create tables in Supabase SQL Editor
Go to Supabase → SQL Editor → run CREATE TABLE commands for all 5 tables. The technical plan has the exact column list for each table.
🧪
How to test Database
3 ways to check everything is saved correctly
🔍
Supabase Table Editor (visual)
Go to Supabase dashboard → Table Editor. You can see all rows in all tables. After a test scan, check if a row appeared in disease_scans. Like looking at an Excel sheet.
💻
SQL Editor
Run SQL queries directly: SELECT * FROM users; — shows all registered users. SELECT * FROM disease_scans ORDER BY scanned_at DESC; — shows latest scans.
📬
Via Postman
Call your Java API endpoints from Postman (e.g. POST /api/auth/register). Then check Supabase Table Editor to confirm a new user row was created.
AI & External APIs — the intelligence layer

These are services your Java backend calls. You don't build them — you just call them like making a phone call and getting an answer back.

🔌
4 External APIs your app uses
🤖
Gemini 1.5 Flash API (Google AI)
The main AI brain. Powers: disease scanner (sends photo), price forecast (sends price history), weather advice (sends weather data), crop advisor (sends soil/water/budget), KisanBot (sends Hindi questions). Get free API key from aistudio.google.com. Java sends text/photo → gets Hindi text back.
📊
Agmarknet API (Indian Government)
Official mandi prices across India. Java calls this with crop name + state → gets back today's real market price in ₹/quintal. Free. Used in Price Forecast + Crop Advisor.
🌤️
OpenWeatherMap API
7-day weather by GPS location. 1 million free calls/month. Java calls with farmer's lat/lng → gets 7-day hourly forecast → sends to Gemini → Gemini converts to farming advice in Hindi.
🎤
Web Speech API (in browser, free)
NOT an API you call from Java — this lives in the React frontend. Built into Chrome/Android browser. No key needed. Mic button → farmer speaks Hindi → browser converts to text → sent to Java. Also: Java reply text → browser speaks it aloud.
